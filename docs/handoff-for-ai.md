# 思い出しあそび — 他の生成AI向けプロジェクト説明書

> このドキュメントは、別の生成AIにコンテキストを渡すためのものです。  
> 最終更新：2026年6月時点の実装状況に基づく。

---

## 1. プロジェクト概要

| 項目 | 内容 |
|---|---|
| **アプリ名** | 思い出しあそび |
| **運営** | 合同会社もなか |
| **テーマ** | 大人があそぶように生きる |
| **サブタイトル** | 1日5分、自分の人生であそぶ |
| **リポジトリ内パス** | `omoide-asobi/`（`monaca/` は別プロジェクトのコーポレートLP） |

### 一言定義

記憶力の低下に悩む大人向けの Web アプリ。**一般的な脳トレ（計算・漢字・IQ競争）ではない。** 自分の人生の思い出（エピソード記憶・感覚記憶）で遊びながら、結果的に脳をやさしく活性化することを目指す。

### 最重要の設計思想（絶対に崩さない）

> **「脳トレをしている感覚」ではなく、「自分の人生で遊んでいる感覚」**

- 正解は一つではない
- 点数・ランキング・正誤判定は **UIに存在しない**
- 思い出せない日は失敗ではなく **「？しおり」** として保存
- 競争より共有、点数より笑い、正解より発見

---

## 2. 作りたくないもの

- 計算問題・漢字問題・間違い探しの IQ 競争
- 「今日のスコアが下がった」という罪悪感
- 認知機能テスト感・脳年齢表示
- 正解一択クイズ・他人とのランキング
- 脳のイラスト・メダル・HPバー・レベル表示

## 4. 画面構成とユーザーフロー

### ルーティング

| パス | 画面 | 説明 |
|---|---|---|
| `/` | ランディング | コンセプト説明・CTA |
| `/login` | ログイン | ゲストではじめる（ニックネーム任意） |
| `/onboarding` | オンボーディング | 3ステップ：世界観→体験問い→しおり体験 |
| `/home` | ホーム | 今日の宝箱・しおり棚・モチの一言 |
| `/play/[sessionId]` | プレイ | 問いに回答（1日2問） |
| `/play/[sessionId]/reflect` | 振り返り | 回答確認・感情色選択・しおり獲得 |
| `/album` | アルバム | 過去の問いと答え（一覧/しおり棚） |
| `/zukan` | あそび人図鑑 | 称号・キーワード・世界地図（入力項目なし） |
| `/settings` | 設定 | ニックネーム・通知設定・宝箱作り直し・ログアウト |

### 1日の体験フロー（5分）

```
ホーム → 宝箱を開ける（アニメ） → 問い1 → 問い2
  → 振り返り → 感情色を選ぶ → しおり獲得 → ホーム
```

- 1日1回、宝箱を開けられる
- 1回あたり **問い2つ**（平日・土曜で構成が異なる、後述）
- 回答は自由記述テキスト（200字程度）
- 「今日は出てこなかった」でスキップ可能（？しおり）

## 5. 問いシステム — 認知刺激の3層モデル

問いは **35問**（`lib/questions.ts`）。`depth` フィールドで3段階に分類。

| 深度 | 名称 | 問い数 | 特徴 | 疲れ方 |
|---|---|---|---|---|
| **①** | あたたかい想起 | 20問 | 自由に思い出す（エピソード・感覚） | ほとんど疲れない |
| **②** | ちょい深度（科学強化） | 10問 | 順番・分類・比較・多感覚を追加 | ほんの少し |
| **③** | 深い宝箱（週1） | 5問 | 切り替え・二重課題・カテゴリ流暢性 | 脳トレより軽い |

### 配信ルール（`lib/daily.ts`）

| 曜日 | 出題構成 |
|---|---|
| **平日（月〜金・日）** | 深度① ×1 ＋ 深度② ×1 |
| **土曜** | 深度③ ×1 ＋ 深度② ×1（**深い宝箱**） |

- 日付をシードにした疑似ランダムで選出
- 直近7日間は同じ問いを出にくい
- 土曜は UI に「✨ 今日は深い宝箱」バッジ、宝箱が金の輪付き

### 深度②の強化例（科学強化版）

| ID | タイトル | 強化内容 |
|---|---|---|
| q01 | 帰り道スケッチ | 3つを **家に近づく順番** で |
| q02 | 教室の座席 | 見えたものを **左→右の順** で |
| q09 | 文化祭・体育祭 | **朝→昼→終わり** の順で3つ |
| q11 | 共通点さがし | 共通する思い出を **2つ以上** |
| q19 | 連想リレー | 5連想→ **最古/最新** の記憶を選ぶ |
| q20 | 匂いで連想 | 匂い＋ **音** を追加 |
| q27 | 食卓の音 | 音と匂いを **分類** |

### 深度③（週1・5問）

| ID | タイトル |
|---|---|
| q31 | 二重の思い出（匂い→音→場所） |
| q32 | 記憶タイムライン（幼少・学生・大人を古い順） |
| q33 | 分類あそび（記憶5つをカテゴリ分け） |
| q34 | 連想スプリント（「夏」から1分連想） |
| q35 | 季節スイッチ（夏→冬に記憶切替） |

### 問いデータ構造（`types/question.ts`）

```ts
type Question = {
  id: string;           // "q01" 〜 "q35"
  title: string;        // 表示名
  body: string;         // 問い本文
  hint: string;         // ヒント（「思い出せなくても大丈夫」等）
  category: "episode" | "discover" | "associate" | "sense";
  senseType?: "smell" | "sound" | "temperature" | "touch" | "sight" | "taste";
  estimatedMinutes: number;
  depth: 1 | 2 | 3;
  cognitiveTargets?: string[];  // 設計用（UIには出さない）
};
```

## 6. キャラクター・世界観

### メインキャラ「モチ」

- 記憶の屋根裏に住むふわふわの生き物
- **先生・コーチではない**。正解を教えない。共感する
- 振り返り画面でキーワードに応じたセリフ（`lib/mochi.ts`）
- 正誤判定のフィードバックは一切しない

### コレクション「思い出しおり」

- 回答した問いがしおりになる
- 感情色：あたたかい / しんみり / おもしろい / ？（スキップ時）
- 深度②以上は箔押し風（太いしおり）
- アルバムで日付別に見返せる

### 宝箱

- 日替わりの見た目：木・布・紙・鉄（土曜は「深い宝箱」で鉄＋金の輪）
- 開封時に CSS アニメーション

### デザイン

- カラー：生成り `#F5F0E8`、木 `#C4A882`、夕暮れ `#E8A87C`、墨 `#3D3630`
- フォント：Noto Sans JP + M PLUS Rounded 1c
- キーワード：紙・木・夕暮れ・手帳・宝箱
- モバイルファースト（max-width 480px）

## 7. データモデル（localStorage）

### 保存キー

| キー | 内容 |
|---|---|
| `omoide_user` | ユーザープロフィール |
| `omoide_sessions` | 日次セッション一覧 |
| `omoide_bookmarks` | しおり一覧 |
| `omoide_play_draft` | プレイ途中データ |

### UserProfile

```ts
{
  id: string;              // "guest_1234567890"
  displayName: string;
  onboardingComplete: boolean;
  reminderEnabled: boolean;
  reminderTime: string;    // "20:00"
  createdAt: string;
}
```

### DailySession

```ts
{
  id: string;              // 日付 "2026-06-10"
  date: string;
  questionIds: string[];   // ["q07", "q19"] など
  answers: Answer[];
  emotion?: "warm" | "nostalgic" | "fun" | "unknown";
  completedAt?: string;
  openedAt?: string;
  isDeepBox?: boolean;     // 土曜の深い宝箱
}
```

### Answer

```ts
{
  questionId: string;
  text: string;
  skipped: boolean;        // true = ？しおり
}
```

### Bookmark

```ts
{
  id: string;
  sessionId: string;
  date: string;
  questionId: string;
  questionTitle: string;
  answerText: string;
  emotion: Emotion;
  skipped: boolean;
  isDeep?: boolean;        // 深度②以上
  createdAt: string;
}
```

### セッション修復

`lib/session-repair.ts` で壊れたセッションデータ（問いIDが空・無効）を自動修復。  
設定画面の「今日の宝箱を作り直す」で手動リセットも可能。

## 12. 他AIへの作業依頼時の注意

1. **思想を最優先** — 「脳トレ感」を出す変更は避ける。点数・正誤・ランキングを追加しない
2. **小さな変更** — 関係ないファイルを書き換えない。既存機能を削除しない
3. **`.env` は触らない** — Firebase 連携時もユーザー明示依頼まで
4. **問いの追加・変更時** — `depth` を必ず設定。深度②は科学強化の要素を含める
5. **データ互換性** — localStorage のキー名を変える場合はマイグレーションを書く
6. **日本語UI** — ユーザーは非エンジニア。説明はやさしい日本語で
7. **モバイルファースト** — スマホで5分遊ぶ想定

## 13. あそび人図鑑（実装済み MVP）

**プロフィールを書かず、遊んだ結果として人らしさが浮かび上がる** 機能。

- 入力項目ゼロ（職業・趣味・自己紹介・MBTI は禁止）
- 思い出しあそびの回答からキーワード・称号を自動蓄積
- ホームに「図鑑」タブ（`/zukan`）
- **実装済み**：localStorage + 辞書マッチング、`/zukan` タブ、セッション完了時に自動更新
- 称号は累計3回答以上で表示、7日クールダウン
- Phase 2 以降：あそび人同士のマッチング（「遊ぶと面白そう」軸）

詳細 → [asobi-zukan-design.md](./asobi-zukan-design.md)
