# 画面構成 — Cursor実装用

> Next.js App Router + Firebase を前提とした画面・ルーティング設計

---

## 設計方針

1. **1日の体験は3タップ以内で開始**（ホーム → 宝箱を開ける → 問い）
2. **点数・ランキング画面は存在しない**
3. **モバイルファースト**（スマホで5分遊ぶ想定）
4. **Client Component は必要最小限**（入力・アニメ・認証）

---

## サイトマップ

```
/                          … ランディング（未ログイン）
/login                     … ログイン・新規登録
/onboarding                … 初回体験（3問サンプル）
/home                      … 今日の宝箱（メイン）
/play/[sessionId]          … 問いに答える
/play/[sessionId]/reflect  … 振り返り・しおり獲得
/album                     … アルバム（過去の問いと答え）
/bookmarks                 … しおり棚（コレクション）
/settings                  … 通知・プロフィール・アカウント
```

MVPでは `/album` と `/bookmarks` は統合してもよい（`/album` にしおり表示を含める）。

## 画面一覧

### 1. ランディング `/`

| 要素 | 内容 |
|---|---|
| ヒーロー | 「1日5分、自分の人生であそぶ」 |
| 説明 | 脳トレではない・正解は一つじゃない・5分 |
| CTA | 「宝箱を開けてみる」→ `/onboarding` or `/login` |
| フッター | もなかについて・プライバシー |

**実装**：Server Component で静的表示

---

### 2. ログイン `/login`

| 要素 | 内容 |
|---|---|
| 入力 | メール＋パスワード or ゲストで始める |
| 文言 | 「あなたの思い出は、あなただけの宝箱に入ります」 |
| 遷移 | 新規 → `/onboarding`、既存 → `/home` |

**実装**：Client Component（Firebase Auth）

---

### 3. オンボーディング `/onboarding`

**3ステップ**

| Step | 画面 | 内容 |
|---|---|---|
| 1 | 世界観 | モチ登場。「ここは記憶の屋根裏です」 |
| 2 | 体験問い | サンプル1問（帰り道）に回答 |
| 3 | しおり体験 | 回答がしおりになる演出 → `/home` へ |

**実装**：Client Component、localStorage で完了フラグ

## 主要フロー（実装順序）

### Phase 1 — 体験が回るまで

1. `lib/questions.ts` に30問を静的データとして定義
2. `/home` で「今日の問い」を日付シードで選ぶ
3. `/play` → `/reflect` の入力と保存（最初は localStorage でも可）
4. 開封アニメーション（CSS）

### Phase 2 — データ永続化

5. Firebase Auth（ゲスト or メール）
6. Firestore に `daily_sessions` 保存
7. `/album` で履歴表示

### Phase 3 — 仕上げ

8. オンボーディング
9. 通知（Web Push or メールは将来）
10. LP `/`

## ルーティングガード

| 条件 | リダイレクト |
|---|---|
| 未ログインで `/home` | → `/login` |
| ログイン済みで `/login` | → `/home` |
| オンボーディング未完了 | → `/onboarding` |
| 今日セッション完了で `/play` | → `/home` or `/reflect` |
